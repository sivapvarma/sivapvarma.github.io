Hello World!

Posted on Aug 15, 2015 in Journal

Over the years, whenever I sat down to create a website, I would get stumped trying to figure out details of hosting. Moreover I always wanted to have a blog to write down stuff so that I could refer back later. Back then in 2010, I looked at Blogger, Wordpress and Tumblr. I did not like them as I had very limited control over the design.

Github Pages provide a very reliable hosting option for static sites at zero cost. As for blogging, there is no dearth of Static site generators these days. It feels like a new one comes and goes everyday just like javascript frameworks do by the hour. I have tried Pelican, Jekyll, and Hakyll.

Pelican is a Python based static site generator. You can install it from PyPI using pip. I spend a lot of time writing Python particularly for Scientific computing, so it was a natural choice for me. At the time there were no good themes for pelican and I did not have the know-how or the time to create one myself. Next I tried , Jekyll.

Jekyll is created by Tom Preston Werner, cofounder of Github. Like all projects officially backed by Github, it looks cool. It has a beautiful website, lots of documentation and loads of good looking themes. I liked Jekyll a lot, especially the fact that I can directly push markdown to my repo rather than the html generated from markdown. However, I never used Ruby. So I did not want to install Ruby, Jekyll and its many dependencies on my computer just for blogging. As of this writing Jekyll is still not officially supported on Windows.

In December 2014, when I was around half way through Learn You A Haskell, I wanted a project to apply my newly acquired haskell knowledge. That’s how I stumbled on Hakyll. Hakyll is very similar to Jekyll, and it worked very well for me on Windows. The trouble started when I tried to reuse source files from Windows on Archlinux. First the infamous Cabal Hell and as though that was not enough when I got everything working, the config files were doing different things on Windows and Linux.

Haskell is a very cool language but currently I am not as comfortable in Haskell as I am in Python1. Eventually I came back to Pelican, when I found a theme of my liking. Pelican’s stability also should help me concentrate on writing the content. I wanted a simple website that will keep the focus on content and I think what I have now is perfect in that respect.

  1. I intend to rectify this, a lot of good software is written in Haskell especially Xmonad, Pandoc and Diagrams, but it seems like I always have a big list of languages to learn. Currently this list includes Haskell, Lisp and Lua. ↩
